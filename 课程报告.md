# TayPoadcast —— 多领域 AI 新闻播客自动生成平台

## 一、项目概述

**项目名称**：TayPoadcast

**所属行业**：AI 音频 / 信息消费

TayPoadcast 是一个全自动的 AI 新闻播客生成平台。它从多个信息源采集热点资讯，利用大语言模型生成 NotebookLM 风格的双人对谈脚本，通过语音合成技术输出 MP3 播客文件，并自动推送到 Telegram、微信、飞书等终端。

项目已完成从技术资讯单领域 MVP 到**多领域模块化平台**的演进，当前内置技术、财经、学术、综合新闻四个领域模块，通过 config.yaml 声明式配置管理。配套的 Flask Web UI 提供了一站式播客控制中心——选择领域、一键生成、在线试听、多通道推送，全部在浏览器中完成，无需命令行操作。项目的核心理念是：**任何领域的热点信息都可以在几分钟内变成可听的播客**，让用户在通勤、健身等碎片化时间高效获取信息。

## 二、核心产品或服务

TayPoadcast 的核心产品是一个**模块化的播客自动生成管线**，包含四个核心层和一个控制层：

1. **信息采集层（Fetcher）**：插拔式 Fetcher 模块架构，每个领域一个独立 Fetcher 类，通过 `FETCHER_REGISTRY` 统一注册。当前已接入四大领域——技术（GitHub Trending + Hacker News）、财经（雪球热帖 + 财联社电报）、学术（arXiv 最新论文）、综合新闻（微博热搜 + 知乎热榜 + 新浪要闻）。所有抓取器实现统一的 `BaseFetcher` 接口，新增领域只需实现 `fetch()` 方法即可接入下游管线。

2. **脚本生成层（Script Generator）**：将热点摘要输入大语言模型，生成双主持人（晓晓 & 云扬）的对话脚本。系统提示词经过精心设计，要求模型做到"深度分析、新旧对比、自然互动"，每期生成 25-35 轮对话、4000-6000 字的口语化内容，并附带仓库中文速览摘要。支持 DeepSeek、Claude、OpenAI 三种 LLM 后端，自动检测可用服务。支持按领域定制提示词（config.yaml 中的 `prompt_extra`），让不同领域的播客具有差异化的解读风格。

3. **语音合成层（Audio Generator）**：使用 Microsoft Edge TTS 免费服务，为两位主持人分配不同的中文原生语音（晓晓：zh-CN-XiaoxiaoNeural，云扬：zh-CN-YunyangNeural），并行生成每句对话的音频片段，再通过 ffmpeg 按发言顺序拼接，自动插入不同说话人之间的停顿间隔和结尾淡出效果，生成自然的双人播客 MP3。

4. **分发推送层（Notifier）**：通过 Telegram Bot API 直发 MP3 音频文件及中文仓库速览；通过微信测试号模板消息推送文字简报；通过飞书机器人 API 发送文本预览和 MP3 文件消息。支持多领域聚合推送——可将当日所有领域播客统一分发，各渠道独立状态跟踪。

5. **Web 控制层（Web UI）**：基于 Flask 的单页应用，采用 **kami（紙）设计系统**——暖色羊皮纸底色（`#f5f4ed`）、墨蓝单色 accent（`#1B365D`）、中文衬线字体排版，呈现日式手帳日志的视觉质感。**左右双栏布局**：左栏固定 340px 放置领域选择卡片、一键生成按钮和推送控制区；右栏弹性宽度承载三标签页（音频播放、文稿阅读、仓库速览）和迷你播放条。飞书凭证支持 Web 端即时填写，无需环境变量。全部操作在浏览器中完成。

## 三、市场分析

### 3.1 用户画像

当前技术资讯模块的核心用户是**软件开发者与 AI 从业者**——他们需要追踪每日涌现的新工具、新框架、新论文，但阅读大量仓库 README 和技术文章的时间成本极高。播客形态允许他们在通勤、运动等场景下"并行消费"信息。

但平台的潜在用户远不止于此。任何需要持续关注行业动态的知识工作者都是目标用户：

- **金融从业者**：需要每日跟踪市场动态、政策变化、公司公告。当前获取方式以文字资讯为主，播客化后可显著降低信息摄入门槛。
- **教育工作者与研究人员**：需要跟踪最新论文、教育政策、学术会议动态。arXiv、PubMed 等信息源可通过相同管线转化为学术播客。
- **普通资讯消费者**：微博热搜、知乎热榜等社会热点同样适合播客化，面向希望利用碎片时间了解社会动态的大众用户。

### 3.2 竞争对手分析及差异化优势

| 对比维度 | NotebookLM (Google) | 传统财经/资讯播客 | Podcastfy (开源) | **TayPoadcast** |
|---------|--------------------|--------------------|-------------------|-----------------|
| 内容来源 | 用户手动上传文档 | 人工采编团队 | 用户提供文本 | 自动抓取多信息源 |
| 生成方式 | AI 对话 + TTS | 人工录制 | AI TTS | AI 全管线自动 |
| 更新频率 | 按需手动 | 固定档期（周更/日更） | 按需 | 每日自动 |
| 运营成本 | 免费 | 极高（人力） | 低 | 极低（~¥0.003/天） |
| 领域覆盖 | 任意文档 | 单一垂直领域 | 任意文本 | **多领域模块化扩展** |
| 分发渠道 | 网页播放 | 各播客平台 | 本地文件 | Telegram + 微信 + 飞书 + GitHub Pages |
| 操作界面 | Web | 无 | CLI | **Web UI + CLI 双模式** |

**核心差异化优势**：

1. **信息源热插拔架构**：与竞品最大的不同在于 TayPoadcast 的信息采集层与下游管线完全解耦。新增一个领域只需实现对应的 Fetcher 模块（继承 `BaseFetcher`，实现 `fetch()` 方法），在 `FETCHER_REGISTRY` 中注册，并在 config.yaml 中添加领域配置。脚本生成和语音合成无需任何改动。这使得边际拓展成本趋近于零。

2. **每日全自动化**：从信息采集到推送的整个流程在 GitHub Actions 上定时运行，用户只需在每天固定时间查收推送即可。这是传统人工播客和按需生成的 AI 工具都不具备的能力。

3. **零运营成本**：LLM 调用选用 DeepSeek（约 ¥0.003/天），Edge TTS 免费，GitHub Actions 免费额度完全够用。一个月总成本不超过 ¥0.1，这使得规模化运营多档垂直播客在经济上完全可行。

4. **国产化适配**：针对国内用户场景，新增飞书推送渠道，微信模板消息推送，以及 Web UI 内飞书凭证即时填写功能（无需配置环境变量即可使用），降低国内用户的使用门槛。

### 4.2 核心功能亮点

**Web UI 播客控制中心**：项目最大的交互升级。基于 Flask 构建的单页应用，无需前端框架依赖，纯原生 JavaScript 实现完整状态管理和 UI 交互。本次迭代从单栏居中布局重写为**左右双栏布局**，并引入 Open Design 社区的 **kami（紙）设计系统**完成视觉升级——暖色羊皮纸底色（`#f5f4ed`）、墨蓝单色 accent（`#1B365D`）、中文衬线字体排版，严格遵循 kami 的排版约束（权重 ≤500、无粗体无斜体、暖灰文字层级、三级 whisper 阴影体系）。特色包括：双栏结构（左选右显，充分利用屏幕横宽）；单领域选择模式；迷你播放条（波形动画 + 倍速切换 + 时间追踪，仅音频 Tab 可见）；1.5 秒轮询近实时进度反馈；飞书凭证即时填写——国产化适配的关键设计，用户无需触碰环境变量即可完成飞书推送。

**多 LLM 后端自动检测**：系统按 DeepSeek → Claude → OpenAI 的优先级自动检测可用 API Key，无需手动指定。推荐 DeepSeek（中文效果好且成本极低），但也支持追求更高质量的 Claude 或兼容 OpenAI 接口的第三方服务。

**双声道自然对话模拟**：通过精心设计的 System Prompt（约 1500 字），指导 LLM 生成具有真实互动感的双人对白——包括打断、抢话、恍然大悟、温和反驳等口语特征。支持按领域定制提示词（config.yaml 中的 `prompt_extra` 字段），例如财经领域强调"宏观经济政策、行业板块轮动"，学术领域强调"通俗语言解释核心创新点"，这层 Prompt 工程是整个播客听感的关键，也是本项目的核心壁垒之一。

**多通道推送**：Telegram 直接发送 MP3 文件（支持 50MB 以内音频），附送 LLM 生成的中文仓库速览作为预览；微信发送文字简报（模板消息），适配微信不支持直接音频推送的限制；飞书发送文本预览 + MP3 文件消息（支持 30MB 以内），采用先上传后发送的两步流程，适配飞书 IM API 规范。

**模块化信息源架构**：当前版本的四领域模块仅是起点。框架天然支持接入更多信息源模块（法律、医疗、能源等垂直行业），详见下文产品规划。

### 4.3 产品规划 —— 多领域模块化拓展

当前版本以四大领域模块覆盖技术、财经、学术、综合新闻场景。本项目的技术框架与具体信息源完全解耦——脚本生成层和语音合成层仅依赖 `DailyDigest` 数据结构，不关心数据来自何处；分发推送层（`notify_multi_domain`）接收统一的领域结果列表，自动遍历推送。这一架构特性使得多领域拓展的边际成本极低。

规划的拓展模块：

- **法律模块**：抓取裁判文书网、法律法规数据库更新，生成"法律资讯速递"播客，面向律师和法务从业者
- **医疗模块**：抓取 PubMed 最新论文、FDA 审批动态，生成"医学前沿"播客，面向医务人员和研究者
- **能源模块**：抓取国家能源局公告、国际油价动态，生成"能源行业早报"播客
- **自定义信息源**：支持用户通过 Web UI 配置 RSS 源、关键词订阅，自动生成个性化播客

多模块之间可独立运行、独立分发，用户按兴趣选择订阅。同一套代码库，不同信息源配置，同时产出多档播客。这是本项目区别于单一领域竞品的核心竞争壁垒。

## 五、商业模式

项目采取**基础免费 + 增值付费**的 Freemium 模式，结合 B2C 和 B2B 双线变现：

### 5.1 B2C 订阅服务

- **免费层**：技术资讯模块（GitHub Trending + HN），每日播客 + 三通道推送（Telegram / 微信 / 飞书），永久免费。作为引流产品，获取种子用户并验证产品价值。
- **付费层（¥9.9/月）**：解锁全部领域模块（财经、学术、综合新闻等），支持自定义信息源（如指定特定 RSS 源、关键词订阅），支持更多 LLM 后端选择（更高音质模型），支持自定义主持人风格。

### 5.2 B2B 企业服务

- **企业内训知识播客**：将企业内部的周报、技术文档、培训材料自动转化为播客，帮助员工利用碎片时间学习。按企业规模年费定价。
- **行业定制播客**：为垂直行业（如医疗、法律、能源）定制专属信息源和脚本风格，以 SaaS 订阅形式交付。
- **白标方案**：为自媒体、咨询公司提供播客自动生成能力，嵌入其现有品牌和分发渠道。

### 5.3 广告与赞助

- **播客内嵌广告**：在免费版播客的结尾插入 15 秒赞助商口播
- **品牌定制模块**：与行业头部品牌合作推出联名播客（如"XX证券 × AI 财经早报"）
